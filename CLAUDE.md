# Claude Code Guidelines for this Repository

## Daily Report Structure

### 構造

```
#### イニシアティブ
- 最上位目標
    - 中間目標
        - 具体的ゴール

#### 実行と振り返り
- [具体的ゴール]
    - 短期目標
    - 進捗と懸念事項

#### 所感
```

### コンセプト

#### 3層のイニシアティブ（Mgr視点のビジョン）
- **最上位目標**: 全社目標に紐づく大きな方針（例：人件費を代替する仕組みを構築する）
- **中間目標**: 最上位目標を実現するための方向性（例：AIで成果を上げられる仕組みを提供する）
- **具体的ゴール**: 中間目標を達成するための具体的な成果物（例：AI SDRの開発）

#### 大葉の概念（手段の柔軟性 vs ゴールの妥協）
CEOのキャンプの例に基づく：
- すばらしい週末を過ごす（最上位目標）
    - キャンプでの最高の一皿（中間目標 / 制約下での価値）
        - ジェノベーゼを作る（具体的ゴール）
            - バジル or 大葉（手段 = 柔軟に対応可）

**重要**: 「大葉」は手段のレイヤー。ゴール以上のレイヤーがブレたら視座が下がっている。

#### 日報が提供する価値（3者への価値）
| 読み手 | 確認できること |
|--------|----------------|
| CEO | イニシアティブのどこに注力したか、ブレていないか |
| 他メンバー | Doiさんの意思決定と注力ポイント |
| 自分自身 | 視座が下がっていないかの振り返り |

#### イニシアティブは毎日変わらなくても良い
- 「イニシアティブ」= 視座を保つためのアンカー
- 毎日見ることで「ここからズレていないか」を確認する
- 日々の変化は「実行と振り返り」で追跡

## Review Guidelines

### Daily Reports and Documents

When reviewing daily reports or other documents, check for the following:

1. **Logic Shifting (論理ずらし)** - References or citations that conflate different concepts (e.g., flexibility in methods vs. lowering standards)
2. **Lowered Perspective (視座の低下)** - Setting "minimum viable" as the goal instead of pursuing excellence
   - Example of what to flag: "As long as hunger is satisfied" (空腹を満たせればいい)
   - Better framing: "Even without the ideal ingredients, we can still create an excellent dish" (最高の一皿を追求する)

### 所感の文章レビュー観点

所感をレビューする際、以下の癖に注意し、該当する場合は指摘する。

1. **同じ自戒の反復になっていないか**
   - 「〜してはいけない」「〜を忘れてはいけない」が過去の所感と同じ内容なら、それは内省ではなく儀式
   - 指摘例: 「この戒めは○月にも書いている。前回から何が変わったかを書いた方が深い」

2. **抽象語で着地していないか**
   - 「構造を捉える」「本質的な価値」「インパクト」で締めると、明日何が変わるか見えない
   - 具体的な行動や判断まで降りているかを確認する

3. **「だろう」が逃げになっていないか**
   - 確信があるのに「〜なのだろう」で留めている箇所は、断定した方がよい
   - 探究の余地を残す「だろう」と、コミットを避ける「だろう」を区別する

4. **「つまり」が不要に多くないか**
   - 「つまり」を削って文が成立するなら不要。言い換えの繰り返しで論が進んでいない可能性

5. **一文が長すぎないか**
   - 3〜4節を接続した複文は分割を検討する

### 内省の深化

所感や振り返りをレビューする際、以下の3つの観点で毎回検討する。
**ただし、些末な指摘は飲み込み、本当に重要なときだけ言及する。**

1. **抜けている視点**: この内省で見落としている角度はないか
2. **学ぶべきこと**: この課題を本気で解くなら、何を知る必要があるか
3. **最も痛烈な批判**: 好意的に読まない人間がこれを読んだら、何と言うか

指摘する基準:
- 同じ自戒の反復に陥っている（内省が儀式化している）
- 重要な利害関係者の視点が完全に欠落している
- 本人が気づいていない前提や思い込みがある
- 表面的な反省で終わっており、構造的な原因に至っていない

### Context

This guideline was established based on CEO feedback (2026-01-21) regarding the importance of:
- Not adopting "trade-off thinking" from referenced materials
- Maintaining high standards while being flexible about methods
- Distinguishing between "flexibility in approach" and "lowering the bar"

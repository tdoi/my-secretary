#### 所感
Claude Code の複数セッションを並列に動かし、別セッションが監督する運用を機能させ始めた。
完全に人が不要なわけではないが、プロジェクトが異なるから、コンテキストが広すぎるからとして、人が思考していた領域を代替させる一歩になったと思う。
その分、全体としての優先度など、意思が必要な部分にリソースを充てていきたい。
ただし、同一プロジェクト内での並列化は引き続き検討が必要。

BPaaS Operation Hub は、単なる機能の集合ではなく、予実を構造で捉えることが重要。
事業計画(予算)に対して案件(実行)を確認し、Lightning Planner の営業プロセスや Deal Rating の案件難易度(予測)と、Momentum Depot・MMP の実際の活動(実行)とのギャップを可視化する。
個別の機能ではなく、予測と実行のギャップを複数の粒度で構造的に捉えることで、リソース調整などの打ち手に繋げられることを意識しなければならない。
